\section{Постановка задачи}

Работа выполняется в рамках открытого исследовательского сообщества \textbf{Lamagraph} (Lambdas, Matrices and Graphs), занимающегося разработкой высокопроизводительных программно-аппаратных решений для анализа данных методами линейной алгебры с использованием функционального программирования. Проект реализуется внутри репозитория \textbf{QTreeFSharp} (Quad-tree based linear algebra in F\# for GraphBLAS-style graph analysis) --- прототипа для реализации сетей взаимодействия (interaction nets).

Фундаментальные графовые алгоритмы (поиск в ширину, подсчет треугольников) требуют эффективных структур для хранения уникальных элементов и проведения массовых теоретико-множественных операций. 

\textbf{Целью работы является} реализация неизменяемой структуры данных \enquote{Множество} на основе сбалансированного АВЛ-дерева на языке F\# и исследование её производительности.

\textbf{Задачи:}
\begin{itemize}
    \item Реализовать структуру АВЛ-дерева и базовые операции (добавление, удаление, поиск) с автоматической балансировкой.
    \item Реализовать теоретико-множественные операции (\verb|union|, \verb|intersection|, \verb|difference|, \verb|symmDifference|).
    \item Разработать альтернативные подходы к операциям над множествами: на основе рекурсивного обхода (\verb|Traversal|) и многопоточности (\verb|ParallelAVLSet|).
    \item Покрыть реализацию Property-based и Unit-тестами.
    \item Провести экспериментальное профилирование посредством BenchmarkDotNet.
\end{itemize}

\section{Предлагаемое решение}

\subsection{Структура данных и обработка ошибок}
Основой решения является тип \verb|AVLSet<'Value>|, представляющий собой неизменяемое АВЛ-дерево. Архитектурной особенностью реализации является отказ от генерации исключений в пользу использования монады \verb|Result| и специализированного вычислительного выражения \verb|resultM|. 

\begin{lstlisting}[caption={Определение базовых типов}]
type AVLSet<'Value> =
    | Empty
    | Node of int * 'Value * AVLSet<'Value> * AVLSet<'Value>

type AVLSetError =
    | RotationError
    | InvalidHeightOfNode
    | EmptyNodeWasNotExpected
\end{lstlisting}

Балансировка при вставке и удалении обеспечивается функциями малых и больших вращений (\verb|LLrotate|, \verb|RRrotate|, \verb|LRrotate|, \verb|RLrotate|). 

\subsection{Алгоритмы теоретико-множественных операций}
Для операций над несколькими множествами спроектированы три различных алгоритмических подхода:

\begin{enumerate}
    \item \textbf{Стандартный (Sequential):} Базовый алгоритм рекурсивного слияния. Опирается на функции \verb|split| (разрезание дерева по ключу) и \verb|join| (слияние деревьев). Обеспечивает логарифмическую сложность и выступает в качестве эталона.
    
    \item \textbf{Оптимизация обходом (Traversal):} Подход для множеств с большой разницей в размере. Алгоритм осуществляет обход меньшего множества и точечное применение операций к большему посредством функции \verb|traverseRes|.
\begin{lstlisting}[caption={Пример реализации объединения через обход}]
let union set1 set2 =
    let maxSet, minSet = Tree.maxMinNodesByHeights set1 set2
    let unSet = Tree.copy maxSet
    Tree.traverseRes Tree.insert unSet minSet
\end{lstlisting}

    \item \textbf{Параллельный (Parallel):} Многопоточная реализация, использующая \verb|Async.Parallel| с настраиваемым ограничением пула потоков. После выполнения разрезания (\verb|split|), левое и правое поддеревья обрабатываются асинхронно в независимых потоках.
\end{enumerate}

Корректность алгоритмов проверялась библиотеками Xunit, FsUnit и FsCheck. Property-тесты доказали сохранение инвариантов АВЛ-дерева (разница высот поддеревьев $\le 1$) после любой последовательности случайных операций.

\section{Экспериментальное исследование производительности}

\subsection{Методика тестирования}
Замеры времени выполнения производились средствами библиотеки \verb|BenchmarkDotNet|. Использовалась следующая аппаратно-программная конфигурация: процессор Intel Core i5-12450H (8 ядер), ОС Ubuntu 24.04.4 LTS, платформа .NET 10.0.7.

Входные данные генерировались случайным образом на этапе инициализации бенчмарка во избежание влияния дискового ввода-вывода. Для одиночных операций размер исходного множества $N$ составлял $100$, $10\,000$ и $100\,000$ элементов. Для бинарных операций размеры множеств $A$ и $B$ варьировались: $A \in \{100, 1000, 10\,000, 100\,000\}$, $B \in \{100, 10\,000\}$.

\subsection{Анализ результатов: Одиночные операции}
Результаты экспериментов подтверждают логарифмическую асимптотику $O(\log N)$. При увеличении размера дерева на три порядка (со 100 до 100\,000 узлов) время выполнения вставки и удаления возрастает $\approx$ в 2,3 раза (с $582$~нс до $1376$~нс). Объём аллоцируемой памяти зависит от текущей высоты дерева и варьируется от 840 до 2080 байт на операцию.

Метод \verb|Traversal| демонстрирует высокую эффективность при пересечении множеств с большой разницей в размерах ($A = 100\,000$, $B = 100$), опережая базовый рекурсивный алгоритм в 3,8 раза ($83,9$~мкс против $318,2$~мкс). При сопоставимых размерах множеств или обратном соотношении (например, $A=100$, $B=10\,000$) применение обхода приводит к существенному снижению производительности (до 41 раза для операции разности).

Для множеств $100\,000 \times 10\,000$ время пересечения составило $16,3$~мс (последовательно) и $363,8$~мс (2 потока). Увеличение времени обусловлено накладными расходами пула потоков (свыше 71\,000 задач на операцию) и нагрузкой на сборщик мусора (аллокация более 100~МБ памяти).

\section{Заключение}

В ходе выполнения работы получены следующие результаты:
\begin{itemize}
    \item Реализована структура данных АВЛ-Множества на языке F\#.
    \item Добавлен API теоретико-множественных операций с поддержкой трех видов алгоритмического исполнения: последовательного, параллельного и основанного на обходе (traversal).
    \item Кодовая база покрыта модульными (Xunit) и property-based (FsCheck) тестами.
    \item Проведено профилирование посредством BenchmarkDotNet. Выявлены границы применимости метода Traversal и доказана избыточность наивной асинхронной многопоточности для подобных рекурсивных вычислений на исследуемых объёмах данных.
\end{itemize}
